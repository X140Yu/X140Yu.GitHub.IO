2015 年的一些计划

2015-01-09 阅读次数 0

终于忙完了期末考试。这篇博文本该出现在元旦，可是很遗憾我拖到了现在。过去的这一年发生了许多事情，印象中是我这一生中过得速度最快的一年。

这期间发生的事情影响我最大的还是编程莫属，虽然我是微电子专业的。从2014年3月8日开始，我学习了许多：从 C 到 C++，从数据结构到算法，从 Windows 到 Linux，仿写过去年很火的 Flappy Bird 和 2048，用 WordPress 和 Jekyll 搭建过博客，看了《黑客与画家》就跑去学 Lisp…可我发现到了年终的时候，自己好像什么都没有学会，没有做过一个像样的，来自自己 idea 的项目。我分析过原因，这可能是学习的深度和广度的原因。去年我学过太多零散的、杂乱无章的知识，很显然，我没有静下心来潜心研究一个方面的知识。就好像你喜欢 Linux，你就一直研究 Linux 相关的知识，他喜欢 Android，他就一门心思开发 Android 应用一样。我太浮躁了。

不过事情也发生过转机，去年11月份左右我发现了 iOS 开发这块新大陆，抱着试一试的心态，没想到一下子就爱上了。初次接触 Objective-C 并没有想象中的那么恐怖，反而觉得用[ ]来调用方法是很可爱的方式，我并没有被这个古老的语言所击败，渐渐的，我找到了自己真正喜欢的东西，那就是开发 iOS 应用程序。终于，这还没有太晚。

2015年对于我来说是至关重要的一年，因为我在今年9月份就要参加传说中的校园招聘，投简历、笔试和面试距离我将不再遥远。我必须抓紧一切时间来提高自身的技术。听过这样一句话，没有量化的目标都是耍流氓，所以我就来指定一下今年的计划：

  • 专注于 iOS 开发，每个月一本（至少） iOS 开发相关书籍，做读书笔记。
  • 每个月至少三篇博文，可以是无关技术的，但是必须要写。
  • 每月至少翻译一篇 iOS 相关的文章，英语不好，何谈 iOS 开发。
  • 2015年3月8日之前，做出一个像样的 iOS 程序来，条件是，自己必须满意。
  • 还是做项目，从项目中学习是捷径。

其实对于我自己来说，计划是一回事，完成计划可能又是另外一回事儿了。希望我2016年再做总结时，不要打自己的脸。
